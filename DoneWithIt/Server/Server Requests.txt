General text request format:
Delimiter list: "~|`"

Message to Server format:
{request_number}~|`{userID/email}~|`{case specific data}

Server response format:
{"success" || "fail"}~|`{relevant data}~|`{...}~|`{request_number if success}
    - Note: every fail will be fail~|`{request_number}
    1. 2
    2. 4
    3. 2
    4. 3
    5. 3
    6. 2
    7. 6
    8. 3
    9. N
    10. 3

1. Check User Exists 
    Format: 1~|`email
    > email

    Format: {"success" || "fail"}~|`1
    < success/fail

2. Add new User
    Format: 2~|`email~|`displayName~|`currentDate
    > email
    > displayName
    > currentDate

    Format: {"success" || "fail"}~|`userID~|`2
    < userID

3. Get existing user data
    Format: 3~|`email
    > email

    Format:
    {"success" || "fail"}~|`userID~|`{profilePic still dont know about pic stuff}~|`{outfit 1 info}~|`{outfit 2 info}Format: {"success" || "fail"}~|`3
    
    < userID
    < profile picture
    < outfit images and associated file paths


4. Update display name
    Format: 4~|`userID~|`displayName
    > userID
    > display name

    Format: {"success" || "fail"}~|`4
    < succes/fail

6. Remove all user data 
    Format: 6~|`userID
    > userID

    Format: {"success" || "fail"}~|`6

7. Add new outfit to database under user
    Format: 7~|`userID~|`{outfit image idk what to do so far}~|`outfitName~|`outfitDescription~|`creationDate
    > user Id
    > outfit image
    > outfit name
    > outfit description

8. Delete outfit from database

    Format: {"success" || "fail"}~|`8

10. Give/receive notification token
    Format: 10~|`userID~|`token/NULL
    > userID
    > token || NULL

    Format: {"success" || "fail"}~|`token~|`10
    < success/failure || token
